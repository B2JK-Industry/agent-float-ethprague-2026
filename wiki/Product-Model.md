# Product Model

## Product Thesis

AI agents should not raise on promises. They should raise after they prove work.

Agent Float creates a clean lifecycle:

1. Agent works.
2. Agent proves work.
3. Builder creates a Umia venture.
4. Agent Float anchors the venture to ENS.
5. Investors fund through Umia.
6. Agent keeps proving work after funding.
7. Builder collateral protects against default.

## What Agent Float Owns

Agent Float owns the proof and accountability surface:

- public agent profile,
- ENS passport,
- receipts feed,
- builder bond,
- milestone registry,
- investor-facing proof UI,
- receipt SDK,
- discovery and leaderboard.

## What Umia Owns

Umia owns the venture/funding surface:

- legal wrapper,
- venture token issuance,
- Tailored Auction,
- noncustodial treasury,
- holder economics,
- governance/decision markets,
- secondary market.

Agent Float must never pitch itself as a replacement for Umia.

## Hard Rule: No Receipts, No Float

An agent cannot be presented as fundable until it has receipts.

Receipts must be:

- on-chain,
- signed by the agent wallet,
- tied to an actual USDC transfer,
- visible from the agent profile,
- verifiable by judges and investors.

## Value Proposition

### For Builders

- Faster access to capital than traditional fundraising.
- Public proof layer for useful agents.
- ENS-native agent identity.
- Credible investor-facing profile.
- Ability to show that funding improved agent productivity.

### For Investors

- More signal than a pitch deck.
- Live receipts feed.
- Builder bond and milestone visibility.
- Umia-native venture/token flow.
- Exit path through Umia secondary market, subject to Umia mechanics.

### For Umia

- Discovery funnel for agentic ventures.
- Quality gate before venture launch.
- Proof layer that improves deal quality.
- Differentiated ETHPrague submission using Umia core products.

## Anti-Positioning

| We Are Not | Why |
|---|---|
| Token launchpad | We do not launch hype tokens without work |
| Agent marketplace | We do not list generic agents or prompts |
| Agent OS | We do not run or orchestrate agents |
| DAO tooling | Governance is Umia-side or existing DAO tooling |
| Trading leaderboard | Extractive, weak Solarpunk fit |

## Business Model

Post-hackathon, Agent Float can become a venture funnel for working agents.

Potential revenue:

- success fee on capital raised,
- premium investor analytics,
- builder onboarding services,
- venture verification tools,
- white-label proof layer for Umia agent ventures.

Near-term hackathon demo should not overfocus on revenue model. The winning story is product clarity and sponsor-native integration.
