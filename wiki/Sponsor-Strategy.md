# Sponsor Strategy

## Primary: Sourcify

Sourcify is core evidence. The demo must show:

- verified vs unverified implementation
- source metadata
- ABI diff
- storage-layout result when available
- direct Sourcify links

Judge line:

> Upgrade Siren turns Sourcify data into a public upgrade-risk alarm.

## Secondary: ENS Most Creative Use

ENS is the contract identity and version map:

- named proxy
- expected implementation
- report pointer
- schema pointer
- live resolution

Judge line:

> ENS becomes a readable upgrade map for protocols, not just a wallet-name lookup.

## Organizer: Future Society

Upgrade Siren is public-good safety tooling:

- helps DAO voters
- helps users
- exposes upgrade risk in public
- does not require private audit access

## Optional: Umia

Only pitch Umia if mentor feedback suggests the Siren Agent due-diligence story is strong.

Do not force Umia if it weakens Sourcify/ENS clarity.
