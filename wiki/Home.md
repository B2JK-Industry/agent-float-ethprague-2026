# Agent Float Wiki

> Agent Float turns working AI agents into investable ventures.

**Tagline:** Your agent has receipts. Now give it runway.

**Hard rule:** No receipts, no float.

Agent Float is the discovery, proof, and accountability layer above Umia ventures. Umia provides the venture wrapper, Tailored Auction, noncustodial treasury, governance/decision-market layer, and secondary market. Agent Float adds ENS-native agent identity, on-chain receipts, builder collateral, and milestone accountability.

This wiki is the human-readable operating manual for the ETHPrague 2026 build.

## One-Screen Explanation

Most AI agent tokens are hype first. Agent Float flips the order:

1. An agent must already perform paid work.
2. The agent receives an ENS passport.
3. The agent emits signed, USDC-cross-validated receipts.
4. The builder creates a Umia venture.
5. Agent Float links the Umia venture to the ENS identity.
6. The builder locks a personal bond.
7. Investors fund through Umia Tailored Auctions.
8. The agent keeps emitting receipts.
9. If the builder defaults, the bond can slash to current Umia venture token holders.

Agent Float does not redefine Umia token economics. Investor exposure follows Umia's venture wrapper.

## Product Sentence

Agent Float is a proof-gated funding layer where working AI agents become Umia ventures only after they show on-chain receipts.
