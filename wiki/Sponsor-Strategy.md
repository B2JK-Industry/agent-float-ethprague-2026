# Sponsor Strategy

## Target Tracks

| Priority | Track | Prize | Why |
|---|---|---|---|
| Primary | Umia Best Agentic Venture | $12K | Funding mechanism and venture wrapper are core |
| Secondary | ENS Most Creative | $2K | ENS passport is structural |
| Bonus | Sourcify | $4K | Verification reinforces open architecture |
| Organizer | Network Economy | Part of ETHPrague $9K organizer pool | Identity + economic coordination + user control |

## Umia

Agent Float should be pitched as a layer above Umia ventures.

Strong points:

- uses Tailored Auctions,
- creates agentic deal flow,
- adds proof gate,
- adds builder accountability,
- makes Umia more investable for working agents.

Risk:

- If we overemphasize fallback contracts, it looks like we are competing with Umia.

Defense:

- Primary docs and demo lead with Umia.
- Conditional/fallback contracts are clearly labeled.

## ENS

ENS is the passport.

Strong points:

- ENSIP-26 records,
- per-agent subnames,
- portable identity,
- verifiable pointers to venture and proof contracts,
- no hardcoded agent identity.

Risk:

- "Metadata in text records" can feel basic.

Defense:

- ENS is not storage decoration; it is the canonical root of agent identity and verification.

## Sourcify

Sourcify is a bonus track, not a primary product dependency.

Strong points:

- every deployed Agent Float contract verified,
- public source links from agent profile,
- open governance proof.

Risk:

- Verification-only may be thin compared to projects using Sourcify datasets.

Defense:

- Keep it lightweight and additive. Do not split attention from Umia/ENS.

## Network Economy

Agent Float fits because it combines:

- identity,
- transparent funding,
- on-chain receipts,
- user-held tokens,
- builder collateral,
- public verification.

Weakness:

- Financialization can look extractive.

Mitigation:

- emphasize proof-first, public goods, transparency, and anti-hype positioning.

## Sponsors To Skip

| Sponsor | Reason |
|---|---|
| SpaceComputer | Hardware path is not natural |
| Apify | Used as infrastructure, not sponsor track unless mentor confirms |
| Swarm | Receipt storage not worth complexity |
